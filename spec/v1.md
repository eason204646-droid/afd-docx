# AFD v1 Specification

AFD (Agent First Document) is a plain-text, line-oriented document format designed for AI Agent manipulation. It bridges the gap between Markdown's simplicity and Office document formats' expressive power.

## 1. File Extension

AFD files use the `.afd` extension.

## 2. Encoding

AFD files MUST be UTF-8 encoded.

## 3. File Structure

An AFD file consists of:

1. **Optional header** — `; AFD v1` format declaration
2. **Optional front matter** — YAML metadata block delimited by `---`
3. **Content body** — Line-oriented content with prefix markers

## 4. Comments

Lines starting with `;` (semicolon followed by space) are comments and MUST be ignored during parsing.

```
; This is a comment
; AFD v1
```

## 5. Front Matter

A YAML metadata block between `---` delimiters at the start of the file.

```yaml
---
title: Document Title
author: Author Name
date: 2025-06-13
---
```

Supported front matter fields:

| Field | Type | Description |
|-------|------|-------------|
| `title` | string | Document title |
| `author` | string | Author name |
| `date` | string | Document date |
| `page.size` | string | Page size (A4, Letter, etc.) |
| `page.margin` | string | Page margin |
| `page.orientation` | string | portrait or landscape |
| `styles` | object | Named style definitions |

## 6. Inline Formatting

AFD reuses CommonMark-style inline formatting:

| Syntax | Result |
|--------|--------|
| `**text**` | Bold |
| `*text*` | Italic |
| `` `text` `` | Code |
| `[text](url)` | Hyperlink |
| `~~text~~` | Strikethrough |
| `{color:red}text{/color}` | Colored text |

## 7. Content Markers

Every content line begins with a prefix marker followed by `: `.

### 7.1 Headings

```
h1: Level 1 Heading
h2: Level 2 Heading
h3: Level 3 Heading
h4: Level 4 Heading
h5: Level 5 Heading
h6: Level 6 Heading
```

### 7.2 Paragraphs

```
p: Paragraph text with **bold** and *italic*.
```

An empty line between `p:` lines creates a new paragraph.

### 7.3 Lists

Unordered lists use `ul:` prefix with `- ` items, terminated by `end`:

```
ul:
- Item one
- Item two with **bold**
- Item three
end
```

Ordered lists use `ol:` prefix with `1. ` items:

```
ol:
1. First item
2. Second item
3. Third item
end
```

Checklists use `cl:` prefix with `[ ] ` or `[x] ` items:

```
cl:
[x] Completed task
[ ] Pending task
end
```

### 7.4 Tables

```
tbl: bordered header
| Col1 | Col2 | Col3 |
| Cell | Cell | Cell |
| Cell | Cell | Cell |
end
```

- `bordered` — draw borders
- `header` — first row is header
- Header separator row (`|---|---|`) is optional (omitted for brevity)

### 7.5 Images

```
img: path/to/image.png
cap: Image caption
w: 80%
pos: center
```

- `img:` — image source path (required)
- `cap:` — caption (optional)
- `w:` — width percentage (optional, default 100%)
- `pos:` — alignment: left, center, right (optional)

### 7.6 Code Blocks

```
code: javascript
console.log("hello");
end
```

The optional language identifier after `code:` enables syntax highlighting.

### 7.7 Page Break

```
br
```

### 7.8 Headers and Footers

```
hdr: Left text | Center text | Right text
ftr: Left text | Center text | Right text
```

Use `|` to separate left, center, and right sections. `@PAGE` inserts the page number.

### 7.9 Raw Content

```
raw: html
<p>Custom HTML content</p>
end
```

## 8. Style Definitions

In front matter:

```yaml
styles:
  h1:
    size: 24
    bold: true
    font: "Inter"
    color: "#1a1a1a"
  normal:
    size: 11
    font: "Inter"
    line-height: 1.5
```

Style properties:

| Property | Type | Example |
|----------|------|---------|
| `size` | number | Font size in points |
| `bold` | boolean | Bold |
| `italic` | boolean | Italic |
| `font` | string | Font family |
| `color` | string | Hex color |
| `line-height` | number | Line spacing multiplier |

## 9. Example

```afd
; AFD v1
---
title: Hello World
author: AI
---

h1: Hello, AFD!
p: This is a **paragraph** with *formatting*.
p: Another paragraph.

ul:
- Item A
- Item B
end

tbl: bordered
| Name | Value |
| Alice | 100 |
| Bob | 200 |
end
```

## 10. Grammar (EBNF)

```
afd         = header? frontmatter? body
header      = "; AFD v1" newline
frontmatter = "---" newline yaml-content "---" newline
body        = { block }
block       = heading | paragraph | list | table | image
            | code-block | page-break | raw-block | comment
heading     = ("h1" | "h2" | "h3" | "h4" | "h5" | "h6") ":" " " text newline
paragraph   = "p:" " " text newline
list        = ("ul:" | "ol:" | "cl:") newline
              { list-item newline }
              "end" newline
list-item   = "- " text | "1. " text | "[x] " text | "[ ] " text
table       = "tbl:" (" bordered"? " header"?)? newline
              { "|" cell { "|" cell } "|" newline }
              "end" newline
image       = "img:" " " path newline [ "cap:" " " text newline ]
              [ "w:" " " percentage newline ]
code-block  = "code:" lang? newline
              { line }
              "end" newline
page-break  = "br" newline
header      = "hdr:" " " text newline
footer      = "ftr:" " " text newline
raw-block   = "raw:" format newline
              { line }
              "end" newline
```
